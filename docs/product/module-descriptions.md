# Module Descriptions

## Onboarding

Creates the initial workspace through backend setup APIs:

- organization
- entity
- module enablement
- initial upload and airlock review

## Control Plane

Provides read surfaces for:

- intents
- jobs
- timeline
- determinism
- snapshots
- lineage
- impact analysis

## Journals

Shows governed journal rows and highlights:

- intent identifiers
- job identifiers
- approval state

This view is a validation-oriented surface. Actions remain available, but visual emphasis is placed on traceability.

## Airlock

Shows governed intake state:

- queue item status
- findings and validation issues
- admit and reject actions

## Reports and Board Packs

Expose downstream evidence surfaces:

- snapshots
- determinism hash
- lineage
- impact analysis

## Shell

The shell keeps four context anchors visible:

- organization
- entity
- module
- period
